Migration From v0.x to v1

‚ö†Ô∏è This is the Tonic UI migration guide for upgrading from v0.x to v1. You have to consider whether you will do the upgrade if the breaking changes are not acceptable to you at the moment. A proper way for the migration is to make v0.x and v1 coexist for a while, then remove v0.x packages once you have finished the migration.

Upgrade Steps

1. Update the dependencies

  • Rename the @trendmicro/react-styled-ui package to @tonic-ui/react
  • Rename the @trendmicro/styled-ui-theme package to @tonic-ui/theme
"dependencies": {
- "@trendmicro/react-styled-ui": "0.x",
- "@trendmicro/styled-ui-theme": "0.x",
+ "@tonic-ui/react": "^1.0.0",
+ "@tonic-ui/theme": "^1.0.0"
}

To make v0.x and v1 coexist, do the following:

"dependencies": {
  "@trendmicro/react-styled-ui": "0.x",
  "@trendmicro/styled-ui-theme": "0.x",
+ "@tonic-ui/react": "^1.0.0",
+ "@tonic-ui/theme": "^1.0.0"
}

You can keep v0.x packages until you are ready to remove them.

2. Update the providers

A TonicProvider is provided by the @tonic-ui/react package. It includes the following providers:

  • ThemeProvider: Provides the theming context for all components.
  • ColorModeProvider: Provides color mode context to all components. Learn more
  • ColorStyleProvider: Provides the color style context to all components. Learn more

Optionally via prop:

  • useCSSBaseline: To omit the recommended CSSBaseline, pass useCSSBaseline={false}.
+ <TonicProvider
+   colorMode={{ defaultValue: 'dark' }}
+   useCSSBaseline={true}
+ >
- <ThemeProvider>
-   <ColorModeProvider value="dark">
-     <ColorStyleProvider>
-       <CSSBaseline />
-       <App />
+   <App />
-     </ColorStyleProvider>
-   </ColorModeProvider>
- </ThemeProvider>
+ </TonicProvider>

See Getting Started to learn more about the providers.

3. Replace deprecated components and Hooks

PseudoBox üí£ (use Box instead)

PseudoBox is deprecated and its props can now be passed to Box directly. Replace all PseudoBox components with Box.

- <PseudoBox
+ <Box
    _hover={{ fontWeight: 'semibold' }}
  >
    Hover Me
- </PseudoBox>
+ </Box>

ToggleSwitch üí£ (use Switch instead)

ToggleSwitch is deprecated and will be removed in the next major release. Use Switch instead.

- <ToggleSwitch />
+ <Switch />

useDisclosure üí£ (use useToggle instead)

useDisclosure is deprecated and will be removed in the next major release. Use useToggle instead.

const [value, toggleValue] = useToggle(false);
toggleValue(); // toggle the value between true and false
toggleValue(true); // set the value to true
toggleValue(false); // set the value to false

withTheme üí£ (use useTheme instead)

withTheme is deprecated and will be removed in the next major release. Use the useTheme Hook instead.

4. Examine breaking changes

The following components changed the display property internally. Check whether you have to do corresponding layout change.

ComponentPreviousCurrentNote
Buttoninline-blockinline-flex
Menuinline-blockinline-flex
SVGIconinline-blockinline-flexIt might affect the layout according to your usage
Spaceinline-blockinline-flexIt might affect the layout according to your usage
Stackinline-blockinline-flexIt might affect the layout according to your usage
Taginline-blockinline-flex
Textinline-blockblockIt might affect the layout according to your usage
- To make inline text, use <Text display="inline" />
- To make inline block text, use <Text display="inline-block" />
- To make inline flex text, use <Text display="inline-flex" />
- Use <Flex direction="row" /> or <Flex direction="column" /> if you need to render multiple components in the same row or column.

System Updates

Theme

Removed dark and light shadows from the theme.shadows object in favor of colorStyle.shadow. See shadows to learn more.

{
  shadows: {
    none: 'none',
-   dark: {
-     sm: '0 2px 8px 0 rgba(0, 0, 0, 0.48), 0 1px 2px 0 rgba(0, 0, 0, 0.16)',
-     md: '0 4px 16px 0 rgba(0, 0, 0, 0.48), 0 2px 4px 0 rgba(0, 0, 0, 0.16)',
-     lg: '0 8px 32px 0 rgba(0, 0, 0, 0.48), 0 4px 8px 0 rgba(0, 0, 0, 0.16)',
-   },
-   light: {
-     sm: '0 2px 8px 0 rgba(0, 0, 0, 0.16), 0 1px 2px 0 rgba(0, 0, 0, 0.08)',
-     md: '0 4px 16px 0 rgba(0, 0, 0, 0.16), 0 2px 4px 0 rgba(0, 0, 0, 0.08)',
-     lg: '0 8px 32px 0 rgba(0, 0, 0, 0.16), 0 4px 8px 0 rgba(0, 0, 0, 0.08)',
-   },
  },
}

You can use useColorStyle Hook to get the shadow value:

const [colorMode] = useColorMode();
const [colorStyle] = useColorStyle({ colorMode });


console.log(colorStyle.shadow.thin);
console.log(colorStyle.shadow.medium);
console.log(colorStyle.shadow.thick);

Accordion

üëâ The Accordion component was fullly reworked to be more declarative and to be more intuitive.

Accordion

  • Removed allowMultiple and allowToggle props in favor of controling state from the parent
  • Removed index and defaultIndex props
  • Removed the onChange prop. Use onToggle on AccordionItem instead
  • Added AccordionToggle, AccordionToggleIcon, and AccordionCollapse components for unstyled toggle and collapse

AccordionItem

  • Renamed the isOpen prop to isExpanded
  • Renamed the defaultIsOpen prop to defaultIsExpanded
  • Renamed the onChange prop to onToggle
  • Renamed the isDisabled prop to disabled

  • Changed the variantColor prop to backgroundColor
