# Usage

You can use any of the components as demonstrated in the documentation. If you are the first time using Tonic UI, you may want to read the following sections to get started.

## Setup provider

For Tonic UI to work, you will need to setup the `TonicProvider` at the root of your application.

Go to the root of your application and do the following:

```jsx
import {
  Box,
  TonicProvider, // Provides theme and context for Tonic UI components
} from '@tonic-ui/react';

function App(props) {
  return (
    <TonicProvider
      colorMode={{
        defaultValue: 'dark', // One of: 'dark', 'light'
      }}
      // Enable these only for standalone applications
      useCSSBaseline // [optional] Applies CSS reset and base styles
      useCSSVariables // [optional] Enables CSS variables
    >
      <Box {...props} />
    </TonicProvider>
  );
}
```

> 📢 **Important:** `useCSSBaseline` and `useCSSVariables` are intended for standalone applications only. If your app is integrated as a widget within another application (i.e., sharing the same document), disable them to avoid conflicts with global styles and CSS variables.

See below for a more complete example:

```jsx
import { Global, css } from '@emotion/react';
import {
  Box,
  PortalManager, // Manages portals in the application
  ToastManager,  // Manages toasts in the application
  TonicProvider, // Provides theme and context for Tonic UI components
  colorStyle,    // Default color style object
  createTheme,   // For theme customization
  useColorMode,  // Hook to manage color mode (e.g., light/dark)
  useColorStyle, // Hook to manage color style
  useTheme,      // Hook to access the theme object
} from '@tonic-ui/react';
import { useEffect } from 'react';

const customTheme = createTheme({
  components: {
    // Set default props for specific components
    //
    // Example:
    // ```
    // 'ToastCloseButton': {
    //   defaultProps: {
    //     'aria-label': 'Close toast',
    //   },
    // },
    // ```
  },
});

function App(props) {
  const colorMode = 'dark'; // Use `localStorage.getItem('uic_theme')` in TrendAI Vision One

  useEffect(() => {
    // Required for CSS theme variables with color mode support
    document.documentElement.setAttribute('data-color-scheme', colorMode);
  }, [colorMode]);

  return (
    <TonicProvider
      colorMode={{
        defaultValue: 'dark', // One of: 'dark', 'light'
      }}
      colorStyle={{
        defaultValue: colorStyle,  // [optional] Not required when using semantic tokens
      }}
      theme={customTheme}
      // Enable these only for standalone applications
      useCSSBaseline // [optional] Applies CSS reset and base styles
      useCSSVariables // [optional] Enables CSS variables
    >
      <ToastManager
        // Ensure that `ToastManager` is positioned above `PortalManager` to allow toast notifications to be displayed within a portal.
        slotProps={{
          transition: {
            sx: {
              '[data-toast-placement^="top"] > &:first-of-type': {
                mt: '4x', // the space to the top edge of the screen
              },
              '[data-toast-placement^="bottom"] > &:last-of-type': {
                mb: '4x', // the space to the bottom edge of the screen
              },
              '[data-toast-placement$="left"] > &': {
                ml: '4x', // the space to the left edge of the screen
              },
              '[data-toast-placement$="right"] > &': {
                mr: '4x', // the space to the right edge of the screen
              },
            },
          },
        }}
      >
        <PortalManager>
          <GlobalStyles />
          <Box
            backgroundColor="background.low" // semantic token
            color="text.primary" // semantic token
            height="100vh"
            {...props}
          />
        </PortalManager>
      </ToastManager>
    </TonicProvider>
  );
}

function GlobalStyles() {
  const [colorMode] = useColorMode();

  // useTheme() automatically resolves color mode tokens (_dark/_light) based on the current color mode.
  // Use theme.get() to look up token values by path.
  const theme = useTheme();

  return (
    <Global
      styles={css`
        :root, :host {
          color-scheme: ${colorMode};
        }
        :focus:not(:focus-visible) {
          outline: none;
        }
        body {
          font-size: ${theme.get('fontSizes.sm')};
          line-height: ${theme.get('lineHeights.sm')};
        }

        ::-webkit-scrollbar {
          width: 8px;
          height: 8px;
        }
        ::-webkit-scrollbar-track {
          background-color: ${theme.get('colors._component.scrollbar.track.enabled')};
        }
        ::-webkit-scrollbar-track:hover {
          background-color: ${theme.get('colors._component.scrollbar.track.hovered')};
        }
        ::-webkit-scrollbar-thumb {
          background-color: ${theme.get('colors._component.scrollbar.thumb.enabled')};
        }
        ::-webkit-scrollbar-thumb:hover {
          background-color: ${theme.get('colors._component.scrollbar.thumb.hovered')};
        }
      `}
    />
  );
}
```

> Learn more about [Color Mode](../../components/color-mode) in the documentation.

> For the **`color-scheme`** CSS property, see [MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/color-scheme) for more information.

> For the **`:focus-visible`** pseudo-class, see [MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/:focus-visible) for more information.

### Applying base CSS styles

> 📢 **Important:** `useCSSBaseline` is intended for standalone applications only. If your app is integrated as a widget within another application (i.e., sharing the same document), do not enable it to avoid conflicts with global styles.

Sometimes you may need to apply base CSS styles to your application. Tonic UI provides an optional `CSSBaseline` component that fixes some inconsistencies across browsers and devices while providing slightly more opinionated resets to common HTML elements. `CSSBaseline` is recommended to add at the root to ensure all components work correctly.

```jsx
<TonicProvider
  useCSSBaseline
>
  <App />
</TonicProvider>
```

### Enabling CSS theme variables

> 📢 **Important:** `useCSSVariables` is intended for standalone applications only. If your app is integrated as a widget within another application (i.e., sharing the same document), do not enable it to avoid conflicts with CSS variables.

Tonic UI supports converting theme tokens defined in the theme to CSS variables. You can enable this behavior by pass `useCSSVariables` to `TonicProvider`:

```jsx
<TonicProvider
  useCSSVariables
>
  <App />
</TonicProvider>
```

When this is enabled, Tonic UI will automatically generate CSS variables for theme color tokens:

```css
:root {
  --tonic-colors-gray-100: #fafafa;
  --tonic-colors-gray-200: #f7f7f7;
  --tonic-colors-gray-800: #1a1a1a;
  --tonic-colors-background-high-light: var(--tonic-colors-gray-100);
  --tonic-colors-background-high-dark: var(--tonic-colors-gray-800);
  --tonic-colors-text-primary-light: var(--tonic-colors-gray-800);
  --tonic-colors-text-primary-dark: var(--tonic-colors-gray-100);
}

[data-color-scheme="dark"] {
  --tonic-colors-background-high: var(--tonic-colors-background-high-dark);
  --tonic-colors-text-primary: var(--tonic-colors-text-primary-dark);
}

[data-color-scheme="light"] {
  --tonic-colors-background-high: var(--tonic-colors-background-high-light);
  --tonic-colors-text-primary: var(--tonic-colors-text-primary-light);
}
```

Components automatically use these CSS variables for consistent styling across the application.

> 📢 **Important:** When `useCSSVariables` is enabled, remember to set `data-color-scheme` on the root element for color mode support. Without it, theme color tokens will not apply correctly.

> **Note:** `useCSSVariables` is not applicable for Shadow DOM. CSS variables injected into `:root` are not accessible inside a shadow root.

```jsx
import { Box, useColorMode } from '@tonic-ui/react';
import { useEffect } from 'react';

function App() {
  const [colorMode] = useColorMode();

  useEffect(() => {
    // Required for CSS theme variables with color mode support
    document.documentElement.setAttribute('data-color-scheme', colorMode);
  }, [colorMode]);

  return (
    <Box>
      {/* Your app content */}
    </Box>
  );
}
```

To understand how CSS theme variables work with color mode, see [CSS theme variables](../../customization/css-theme-variables/) for details.

### Extending the color style

Optionally, you can extend the color style by providing a custom color style. This is not required if you are using the default color style.

#### 1. Customizing the color style

To override default color style, you can create a color style object according to the color mode, and pass the object to `TonicProvider` with either the `value` or the `defaultValue` property.

```js
import { colorStyle } from '@tonic-ui/react';
import { merge } from '@tonic-ui/utils';

// Let's say you want to define color style for your components
const customColorStyle = merge(colorStyle, {
  dark: {
    risk: {
      high: 'alert.riskLevel.high.text',
      medium: 'alert.riskLevel.medium.text',
      low: 'alert.riskLevel.low.text',
      none: 'alert.riskLevel.uta.text',
    },
    severity: {
      critical: 'alert.severity.critical.text',
      high: 'alert.severity.high.text',
      medium: 'alert.severity.medium.text',
      low: 'alert.severity.low.text',
      info: 'alert.severity.informational.text',
    },
  },
  light: {
    risk: {
      high: 'alert.riskLevel.high.text',
      medium: 'alert.riskLevel.medium.text',
      low: 'alert.riskLevel.low.text',
      none: 'alert.riskLevel.uta.text',
    },
    severity: {
      critical: 'alert.severity.critical.text',
      high: 'alert.severity.high.text',
      medium: 'alert.severity.medium.text',
      low: 'alert.severity.low.text',
      info: 'alert.severity.informational.text',
    },
  },
});
```

#### 2. Setting up the provider

```jsx
<TonicProvider
  colorStyle={{
    defaultValue: customColorStyle,
  }}
>
  {children}
</TonicProvider>
```

#### 3. Using the color style

```jsx
const [colorStyle] = useColorStyle()
console.log(colorStyle?.risk?.medium); // 'alert.riskLevel.medium.text'
console.log(colorStyle?.severity?.critical); // 'alert.severity.critical.text'
```

### Extending the theme

Optionally, you can extend the theme object to override the defaults with custom colors, fonts, styles, etc. This is not required if you are using the default theme.

#### 1. Overriding the theme

You can create a custom theme with primitive tokens, semantic tokens, and other customizations:

```js
import { createTheme } from '@tonic-ui/react';

const customTheme = createTheme({
  colors: {
    // Primitive tokens
    gray: {
      100: '#fafafa',
      200: '#f7f7f7',
      800: '#1a1a1a',
    },
    
    // Semantic tokens (use references that will be resolved based on color mode)
    background: {
      high: {
        _light: '{gray.100}',
        _dark: '{gray.800}',
      },
    },
    text: {
      primary: {
        _light: '{gray.800}',
        _dark: '{gray.100}',
      },
    },
  },
});
```

#### 2. Setting up the provider

```jsx
<TonicProvider
  theme={customTheme}
>
  {children}
</TonicProvider>
```

#### 3. Using theme tokens

Use semantic tokens directly in component props:

```jsx
<Box backgroundColor="background.low" color="text.primary">
  Content with semantic tokens
</Box>
```

> 💡 **NOTICE**<br/>
> You usually don't need `useTheme()` at all — pass semantic tokens like `"text.primary"` or `"background.low"` directly as style props. Use `useTheme()` only when you need to compose resolved values into expressions like `calc()`, `color-mix()`, `box-shadow`, or `linear-gradient`.

The theme object returned by `useTheme()` includes a `get()` method for resolving nested token values by path. `theme.get()` automatically extracts the `main` value from primitive color objects that contain lighten/darken variants.

```jsx
import { useTheme } from '@tonic-ui/react';

function MyComponent() {
  const theme = useTheme();

  // Resolve a semantic color token
  const focusRingColor = theme.get('colors._component.keyboardFocused.innerFocusRing');

  // Resolve a primitive color — automatically extracts `main` from nested objects
  // Both return '#1362fc' (the main value of blue.600)
  const blue600 = theme.get('colors.blue.600');

  // Access lighten/darken variants directly
  const blue600Darken80 = theme.get('colors.blue.600.darken.80');

  // Supports default values for missing paths
  const fallback = theme.get('colors.nonexistent', '#000000');

  // Non-color values work too
  const size = theme.get('sizes.4x');

  // ...
}
```

To resolve tokens for a specific color mode regardless of the current context, pass the `colorMode` option:

```jsx
// Always resolves tokens for light mode, regardless of the active color mode
const theme = useTheme({ colorMode: 'light' });
const lightBackground = theme.get('colors.background.high');
```

#### Shorthands

There are some shorthands available for various CSS properties. These are documentated on the [style props](../../styled-system/style-props) page.

```jsx
<Box
  bg="background.high" // backgroundColor: 'background.high'
  mb="3x" // marginBottom: '3x'
  px="4x" // paddingLeft: '4x', paddingRight: '4x'
  py="3x" // paddingTop: '3x', paddingBottom: '3x'
/>
```

These shorthands are optional. It's great for saving time, but not necessary to use.

### Superset of CSS

The `sx` prop supports CSS syntax including child and pseudo-selectors, media queries, raw CSS values, and more. Here are a few examples of how you can implement these CSS features:

* Using pseudo-selectors:

  ```jsx
  <Box
    sx={{
      ':hover': {
        backgroundColor: 'background.high',
      },
    }}
  />
  ```

* Using media queries:

  ```jsx
  <Box
    sx={{
      '@media print': {
        width: 300,
      },
    }}
  />
  ```

* Using nested selectors:

  ```jsx
  <Box
    sx={{
      '& .child-selector': {
        backgroundColor: 'background.high',
      },
    }}
  />
  ```

Read more on [the sx prop](../the-sx-prop) page.
