# Highlight

The `Highlight` component is used to highlight text that matches a search query. It supports multiple search terms, case-sensitive/insensitive matching, and accent-insensitive matching.

## Import

```js
import { Highlight } from '@tonic-ui/react';
```

### Using the `transform` function

The `transform` function allows you to transform search words before matching. This enables custom matching behavior such as accent-insensitive search, case normalization, or any other text transformation needed for your use case.

```js
const transformAccents = (x) => {
  if (typeof x !== 'string') {
    return x;
  }
  // Convert to NFD (base letters + combining marks), then strip diacritics
  return x.normalize('NFD').replace(/[\u0300-\u036f]/g, '');
};
```

## Props

### Highlight

| Name | Type | Default | Description |
| :--- | :--- | :------ | :---------- |
| children | ReactNode | | The content to search and highlight. It can be a plain string or React elements containing text. |
| caseSensitive | boolean | false | Whether matching should be case-sensitive. |
| query | string \| string[] | | The search word(s) to highlight within the content. |
| slots | object | | Custom components used for rendering. Use `slots.mark` to replace the default `Mark` component. |
| slotProps | object | | Props forwarded to slot components. Use `slotProps.mark` to pass additional props to the `Mark` component. |
| transform | function | | A function that transforms each search word before matching. Useful for implementing custom matching behavior such as accent-insensitive search, text normalization, or other preprocessing. |
| variant | string | 'highlight' | The visual style applied to highlighted text. Passed to the `Mark` component. Supported variants: 'highlight', 'emphasis'. |
