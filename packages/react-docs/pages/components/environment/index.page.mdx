# Environment

The `EnvironmentProvider` enables Tonic UI components to work correctly in non-standard DOM environments such as iframes, Shadow DOM, or Electron applications.

## Why do I need this?

**TL;DR:** If your app renders in a standard browser environment, you don't need to configure anything. The environment context is automatically provided by default.

Many Tonic UI components internally use DOM APIs like `document.querySelector()` or `window.getComputedStyle()`. In standard environments, these work fine. However, in special contexts like iframes or Shadow DOM, the global `document` and `window` objects don't reference the correct context, causing components to malfunction.

The `EnvironmentProvider` solves this by providing the correct DOM context to all components.

## When do I need to configure this?

You need to configure the environment when rendering your application in:

- **Shadow DOM** - Components are rendered inside a shadow root
- **iframes** - Components are rendered inside an iframe's document
- **Electron apps** - When using multiple browser windows
- **Web Components** - When building custom elements with isolated DOM trees

For standard browser applications, the default configuration works out of the box.

## Setup

By using `TonicProvider` at root of your application, it will add `EnvironmentProvider` internally to provide the environment context to all components.

To get environment working correctly in non-standard DOM contexts, you need to pass the `environment` config to `TonicProvider` with `value` set to your custom root node.

```jsx
import React from 'react';
import { Box, TonicProvider } from '@tonic-ui/react';

function App(props) {
  return (
    <TonicProvider
      /**
       * The `environment` config
       * @param {RootNode | (() => RootNode)} value The root node or a function that returns the root node.
       */
      environment={{
        value: document, // or `() => document`
      }}
    >
      <Box {...props} />
    </TonicProvider>
  );
}
```

The above setup is equivalent to:

```jsx
<EnvironmentProvider value={document}>
  <Box {...props} />
</EnvironmentProvider>
```

**Note:** Use a function `() => rootNode` when the root node might change or isn't available during initial render.

## Iframe vs Shadow DOM

Both contexts require an `environment` config. Shadow DOM also needs extra setup for CSS-in-JS scoping and CSS variables.

| | Iframe | Shadow DOM |
| :--- | :----- | :--------- |
| `environment` value | `iframe.contentDocument` | shadow root (`ShadowRoot`) |
| Emotion cache `container` | `iframe.contentDocument.head` | shadow root |
| CSS variables root | not needed (defaults to `:root`) | `':host'` — scopes CSS variables to the shadow host |
| Full setup guide | — | [Shadow DOM](../../customization/shadow-dom) |

### Usage with Shadow DOM

Set `environment` to the shadow root. Portal-based components then render within the shadow DOM boundary. See [Shadow DOM](../../customization/shadow-dom) for the complete setup, including `createShadowDOM`, cache, and theme configuration.

```jsx
const cache = createCache({
  key: 'tonic-css',
  nonce: 'your-nonce-value',
  prepend: true,
  container: shadowRoot,
});

const shadowTheme = createTheme({
  cssVariables: {
    prefix: 'tonic',
    rootSelector: ':host',
  },
});

<CacheProvider value={cache}>
  <TonicProvider
    environment={{ value: shadowRoot }}
    theme={shadowTheme}
  >
    <App />
  </TonicProvider>
</CacheProvider>
```

### Usage with iframe

Set `environment` to the iframe's document. Portal-based components then render within the iframe context.

```jsx
const cache = createCache({
  key: 'tonic-css',
  nonce: 'your-nonce-value',
  prepend: true,
  container: iframe.contentDocument.head,
});

const theme = createTheme({
  cssVariables: {
    prefix: 'tonic',
    rootSelector: ':root', // ':root' works correctly in the iframe's own document
  },
});

<CacheProvider value={cache}>
  <TonicProvider
    environment={{ value: iframe.contentDocument }}
    theme={theme}
  >
    <App />
  </TonicProvider>
</CacheProvider>
```

## Why the environment matters in an iframe

An iframe is a separate realm: its `document` and `window` are different objects from the page's global `document` and `window`. Since Tonic UI components render into the iframe from the parent page's JavaScript, the global `document`/`window` point at the parent page — the wrong realm. Configuring the `environment` lets `useEnvironment()` resolve to the iframe's own `document` and `window`. ## Using `useEnvironment` hook

The [useEnvironment](./useEnvironment) hook provides access to the correct `document` and `window` objects for the current environment. Use it when you need to query DOM elements, add event listeners, measure elements, or create portals in your custom components.

```jsx
const { getRootNode, getDocument, getWindow } = useEnvironment();
```

| | Standard browser | Shadow DOM | iframe |
| :-- | :-- | :-- | :-- |
| `getRootNode()` | `document` | `ShadowRoot` | `iframe.contentDocument` |
| `getDocument()` | `document` | `shadowRoot.ownerDocument` | `iframe.contentDocument` |
| `getWindow()` | `window` | `shadowRoot.ownerDocument.defaultView` | `iframe.contentWindow` |

## Props

### EnvironmentProvider

| Name | Type | Default | Description |
| :--- | :--- | :------ | :---------- |
| children | ReactNode | | The content of the component. |
| value | RootNode \| (() => RootNode) | `document` | The root node or a function that returns the root node. Can be a `Document`, `ShadowRoot`, or any `Node`. Use a function when the node might not be available during initial render. |

### useEnvironment

The `useEnvironment` hook returns an object with the following methods:

| Method | Return Type | Description |
| :----- | :---------- | :---------- |
| getRootNode | () => RootNode | Returns the configured root node — `document` in a standard browser, `ShadowRoot` in a shadow DOM, or `iframe.contentDocument` in an iframe. |
| getDocument | () => Document | Returns the `Document` for the current environment — `document` in a standard browser, `shadowRoot.ownerDocument` in a shadow DOM, or `iframe.contentDocument` in an iframe. |
| getWindow | () => Window | Returns the `Window` for the current environment — `window` in a standard browser, `shadowRoot.ownerDocument.defaultView` in a shadow DOM, or `iframe.contentWindow` in an iframe. |
