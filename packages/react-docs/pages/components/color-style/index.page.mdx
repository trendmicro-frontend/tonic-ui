# Color style

Tonic UI comes with a color style system that defines functional color values.

## Overview

Starting in Tonic UI v4, the color style system is backed by [semantic tokens](../../theme/colors/#semantic-tokens). The API remains unchanged — color style values now map to their semantic token equivalents:

| Semantic token | Color style |
| :--- | :--- |
| <ColorPreview value="background.low" /> `background.low` | `colorStyle.background.primary` |
| <ColorPreview value="background.high" /> `background.high` | `colorStyle.background.secondary` |
| <ColorPreview value="background.highest" /> `background.highest` | `colorStyle.background.tertiary` |
| <ColorPreview value="background._inverse.low" /> `background._inverse.low` | `colorStyle.background.inverted` |
| <ColorPreview value="actions.hovered" /> `actions.hovered` | `colorStyle.background.highlighted` |

## Setup

By using `TonicProvider` at root of your application, it will add `ColorStyleProvider` internally to provide the color style context to all components. The default color style will be added automatically.

```jsx
import { Box, TonicProvider } from '@tonic-ui/react';
import { colorStyle } from '@tonic-ui/react';

function App(props) {
  return (
    <TonicProvider
      /**
       * The `colorStyle` config
       * @param {object} defaultValue
       * @param {object} value
       * @param {function} onChange(colorStyle)
       */
      colorStyle={{
        // [Optional] No need to pass default color style if there's no customization
        defaultValue: colorStyle,
      }}
    >
      <Box {...props} />
    </TonicProvider>
  );
}
```

The above setup is equivalent to:

```jsx
<ColorStyleProvider>
  <Box {...props} />
</ColorStyleProvider>
```

### Color style settings

To get the default color style settings, import `colorStyle` as shown below:

```js
import { colorStyle } from '@tonic-ui/react';
```

The full configuration is displayed below:

<PreformattedText mb="4x">
  {jsonPrettify(colorStyle)}
</PreformattedText>

To manage color style in your application, you can use the [useColorStyle](../../components/color-style/useColorStyle) hook.

### Overriding color style

To override default color style, you can create a color style object in accordance to the color mode, and pass the object to `TonicProvider` with either the `value` or the `defaultValue` property.

```jsx
import { Box, TonicProvider, colorStyle, useColorStyle } from '@tonic-ui/react';
import { merge } from '@tonic-ui/utils';

const customColorStyle = merge(colorStyle, {
  dark: {
    risk: {
      high: 'riskLevel.high.text',
      medium: 'riskLevel.medium.text',
      low: 'riskLevel.low.text',
      none: 'riskLevel.uta.text',
    },
    severity: {
      critical: 'severity.critical.text',
      high: 'severity.high.text',
      medium: 'severity.medium.text',
      low: 'severity.low.text',
      info: 'severity.informational.text',
    },
  },
  light: {
    risk: {
      high: 'riskLevel.high.text',
      medium: 'riskLevel.medium.text',
      low: 'riskLevel.low.text',
      none: 'riskLevel.uta.text',
    },
    severity: {
      critical: 'severity.critical.text',
      high: 'severity.high.text',
      medium: 'severity.medium.text',
      low: 'severity.low.text',
      info: 'severity.informational.text',
    },
  },
});

const App = () => (
  <TonicProvider
    colorStyle={{
      defaultValue: customColorStyle,
    }}
  >
    <Box {...props} />
  </TonicProvider>
);
```
