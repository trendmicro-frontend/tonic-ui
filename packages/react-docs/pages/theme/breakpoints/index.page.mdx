# breakpoints

The default breakpoints is an array of the following values:

```js
[
  '320px',  // sm
  '744px',  // md
  '1440px', // lg
  '1680px', // xl
  '1920px', // 2xl
]
```

The first value in the array is the smallest breakpoint, and the last value is the largest. You can also use `sm`, `md`, `lg`, `xl`, and `2xl` as breakpoint names to reference the breakpoints in the array.

| Breakpoint | Media Query |
| :--------- | :---------- |
| `sm` | `@media screen and (min-width: 320px)` |
| `md` | `@media screen and (min-width: 744px)` |
| `lg` | `@media screen and (min-width: 1440px)` |
| `xl` | `@media screen and (min-width: 1680px)` |
| `2xl` | `@media screen and (min-width: 1920px)` |

## Customizing breakpoints

For more control over the breakpoints, you can define custom breakpoints for your application.

To do this, add a `breakpoints` array with additional aliases (e.g. `sm`, `md`, `lg`, `xl`, and `2xl`) to your theme.

```jsx
// 1. Import the theme
import {
  TonicProvider, // Provides theme and context for Tonic UI components
  createTheme,   // For theme customization
} from '@tonic-ui/react';

// 2. Update the breakpoints
const breakpoints = [
  '320px',
  '744px',
  '1440px',
  '1680px',
  '1920px',
];
breakpoints.sm = breakpoints[0];
breakpoints.md = breakpoints[1];
breakpoints.lg = breakpoints[2];
breakpoints.xl = breakpoints[3];
breakpoints['2xl'] = breakpoints[4];

// 3. Extend the theme
const customTheme = createTheme({
  breakpoints,
});

// 4. Pass the custom theme to the theme provider
function App() {
  return (
    <TonicProvider theme={customTheme}>
      {children}
    </TonicProvider>
  );
}
```
