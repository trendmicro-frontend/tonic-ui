# useId

A custom Hook that generates a unique ID string for accessibility and server-side rendering.

## Overview

Common Use Cases:
- Form elements: Associate labels with inputs
- ARIA attributes: Link elements with `aria-labelledby`, `aria-describedby`
- Component instances: Provide unique identifiers when multiple instances are rendered

Best Practices:
- Use to create accessibility relationships between elements
- Avoid using as React keys in lists (prefer stable data keys instead)
- Generate separate IDs for each relationship you need

## Import

```js
import { useId } from '@tonic-ui/react-hooks';
```

## Usage

```js
const id = useId();
```

### Returns

Returns a unique ID string. May return `undefined` during SSR with React < 18.

* For React 18+:<br/>
  Uses React's built-in `useId()` and always returns the same deterministic string on both server and client.

* For React < 18:<br/>
  Falls back to a custom implementation that may return `undefined` on initial server render to prevent hydration mismatches.
