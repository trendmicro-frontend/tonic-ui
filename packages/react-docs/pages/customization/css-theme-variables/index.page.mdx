# CSS theme variables

Learn how to adopt CSS theme variables.

## Getting started

CSS theme variables are generated by default in `createTheme()`. To enable them at runtime, pass `useCSSVariables` to `TonicProvider`:

```jsx
import {
  TonicProvider,
  createTheme,
} from '@tonic-ui/react';

const theme = createTheme();

function App() {
  return (
    <TonicProvider
      theme={theme}
      useCSSVariables
    >
      {/* Your app content */}
    </TonicProvider>
  );
}
```

After rendering, you will see all CSS theme variables in the `:root` stylesheet of the HTML document. By default, these variables are prefixed with `--tonic`. You can open the **Developer Tools** and go to the **Elements** tab to see all the CSS theme variables being used on the webpage.

> **Note:** CSS theme variables are optional. If you don't need them, simply omit the `useCSSVariables` prop from `TonicProvider`.

## Setting up color mode with CSS variables

**Important:** Without setting the `data-color-scheme` attribute on the correct root element, the color mode-specific CSS variables (those with `_dark` and `_light` variants) will not be applied correctly, and your components may not display the correct colors for the current color mode.

When using CSS theme variables with color mode support (dark/light mode), you need to set the `data-color-scheme` attribute on the appropriate root element. Tonic UI generates CSS variables with selectors like `[data-color-scheme="dark"]` and `[data-color-scheme="light"]` to apply the appropriate color values for each mode.

**For regular DOM:**

```jsx
import { Box, useColorMode } from '@tonic-ui/react';
import { useEffect } from 'react';

function App() {
  const [colorMode] = useColorMode();

  useEffect(() => {
    // Set the data-color-scheme attribute to enable color mode-specific CSS variables
    document.documentElement.setAttribute('data-color-scheme', colorMode);
  }, [colorMode]);

  return (
    <Box>
      {/* Your app content */}
    </Box>
  );
}
```

**For shadow DOM:**

```jsx
import { Box, useColorMode } from '@tonic-ui/react';
import { useEffect, useRef } from 'react';

function App() {
  const [colorMode] = useColorMode();
  const containerRef = useRef();

  useEffect(() => {
    // For shadow DOM, set the attribute on the shadow host
    const shadowRoot = containerRef.current?.getRootNode();
    if (shadowRoot instanceof ShadowRoot) {
      shadowRoot.host.setAttribute('data-color-scheme', colorMode);
    }
  }, [colorMode]);

  return (
    <Box ref={containerRef}>
      {/* Your app content */}
    </Box>
  );
}
```

## Customizing variable prefix

You can customize the variable prefix by providing a string to the `prefix` property in the theme configuration.

```js
createTheme({
  cssVariables: {
    prefix: 'custom',
  },
});
```

```css
:root {
  --custom-borders-1: .0625rem solid;
  --custom-borders-2: .125rem solid;
  /* More variables */
}
```

If you prefer not to use any prefix, set `prefix` to an empty string:

```js
createTheme({
  cssVariables: {
    prefix: '',
  },
});
```

```css
:root {
  --borders-1: .0625rem solid;
  --borders-2: .125rem solid;
  /* More variables */
}
```

## Variables inside the shadow DOM

To apply CSS theme variables inside shadow DOM, specify a different root selector using the `rootSelector` option:

```js
createTheme({
  cssVariables: {
    rootSelector: ':host',
  },
});
```

```css
:host {
  --tonic-borders-1: .0625rem solid;
  --tonic-borders-2: .125rem solid;
  /* More variables */
}
```
