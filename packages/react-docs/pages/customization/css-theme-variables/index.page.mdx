# CSS theme variables

Learn how to generate and consume `--tonic-*` CSS custom properties at runtime.

## Getting started

CSS theme variables are generated by `createTheme()` when `cssVariables` is enabled, and injected into the document by passing `useCSSVariables` to `TonicProvider`:

```jsx
import {
  TonicProvider,
  createTheme,
} from '@tonic-ui/react';

const theme = createTheme({ cssVariables: true });

function App() {
  return (
    <TonicProvider theme={theme} useCSSVariables>
      {/* Your app content */}
    </TonicProvider>
  );
}
```

`useCSSVariables` on `TonicProvider` does two things:

1. Injects a `<CSSVariables />` component that writes `--tonic-*` variable definitions into the document.
2. Instructs the styled-system to emit `var(--tonic-*)` references in component styles instead of raw token values, so live CSS variable overrides take effect immediately.

After rendering, the variables appear in the `:root` stylesheet of the HTML document. Open **Developer Tools → Elements** to inspect them.

### Token naming

Tonic UI uses colon-delimited color token names (e.g. `gray:100`). When converted to CSS variables, colons and dots are replaced with hyphens:

```
theme.colors['gray:100']  →  --tonic-colors-gray-100
theme.colors['blue:50']   →  --tonic-colors-blue-50
theme.fontSizes.sm        →  --tonic-font-sizes-sm
```

You can use these variables in any CSS context — inline styles, `@emotion/css`, or plain stylesheets — without importing the theme object:

```css
.my-element {
  background-color: var(--tonic-colors-gray-100);
  font-size: var(--tonic-font-sizes-sm);
}
```

### Legacy setup (pre-v2.5.0)

If you are using a version prior to v2.5.0 that does not have `createTheme`, use:

```jsx
import {
  TonicProvider,
  theme,
} from '@tonic-ui/react';

const customTheme = {
  ...theme,
  config: {
    ...theme?.config,
    useCSSVariables: true,
  },
};

function App() {
  return (
    <TonicProvider theme={customTheme}>
      {/* Your app content */}
    </TonicProvider>
  );
}
```

## Customizing variable prefix

You can customize the variable prefix by providing a string to the `prefix` property in the theme configuration.

```js
createTheme({
  cssVariables: {
    prefix: 'custom',
  },
});
```

```css
:root {
  --custom-borders-1: .0625rem solid;
  --custom-borders-2: .125rem solid;
  /* More variables */
}
```

If you prefer not to use any prefix, set `prefix` to an empty string:

```js
createTheme({
  cssVariables: {
    prefix: '',
  },
});
```

```css
:root {
  --borders-1: .0625rem solid;
  --borders-2: .125rem solid;
  /* More variables */
}
```

## Variables inside the shadow DOM

To apply CSS theme variables inside a shadow DOM tree, emit them on the shadow host rather than `:root`:

```js
createTheme({
  cssVariables: {
    rootSelector: ':host',
  },
});
```

```css
:host {
  --tonic-borders-1: .0625rem solid;
  --tonic-borders-2: .125rem solid;
  /* More variables */
}
```

See [Shadow DOM](../shadow-dom) for complete setup patterns.
