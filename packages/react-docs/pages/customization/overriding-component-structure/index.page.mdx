# Overriding component structure

Many components are composed of smaller internal parts. The `slots` and `slotProps` props give you granular control over those parts—letting you replace the element rendered for a part, or pass extra props to it—without forking the component.

This is the standardized replacement for the legacy `*Component` / `*Props` prop pairs (such as `TransitionComponent` / `TransitionProps` and `PopperComponent` / `PopperProps`), which are deprecated and will be removed in a future release. Those props were applied inconsistently across components; `slots` and `slotProps` make the API consistent and predictable.

## Overview

A component's structure is determined by the elements that fill that component's **slots**. Slots are most commonly filled by HTML tags, but may also be filled by React components.

- Every component has a **root slot** that defines its primary node in the DOM tree.
- More complex components also contain **interior slots**, named after the parts they represent—for example, `ModalContent` exposes a `transition` slot and a `closeButton` slot, while `TooltipContent` exposes `popper`, `transition`, and `arrow`.

You customize a slot in two independent ways:

- **`slots`** — replace the element rendered for a slot with your own component.
- **`slotProps`** — pass additional props to the element rendered for a slot.

> Be mindful of the rendered DOM structure when overriding the slots of more complex components. Replacing a slot with an element that produces different markup can break semantic or accessible HTML, so prefer `slotProps` (configuring the default element) over `slots` (replacing it) unless you specifically need different rendering.

## Passing props to a slot

Use `slotProps` to forward props to a slot's element without replacing it. A common use case is adding a `data-test` attribute to a close button for UI automation:

```jsx
// Alert, Tag, and Toast — set slotProps on the component
<Alert
  isClosable
  onClose={onClose}
  slotProps={{
    closeButton: { 'data-test': 'alert-close-button' },
  }}
/>

// Modal and Drawer — set slotProps on ModalContent / DrawerContent
<ModalContent
  isClosable
  onClose={onClose}
  slotProps={{
    closeButton: { 'data-test': 'modal-close-button' },
  }}
>
  ...
</ModalContent>
```

The slot's default element is preserved; your props are merged into it.

## Replacing a slot element

Use `slots` when you need to replace a slot's element entirely—both its rendering and behavior—with your own component:

```jsx
<ModalContent
  slots={{ transition: Fade }}
  slotProps={{ transition: { appear: false } }}
/>
```

`slots` and `slotProps` are independent: pass props to the default element with `slotProps` alone, or combine both to replace an element and configure it.

## The `root` slot

Where a component exposes a `root` slot, you can customize its outermost element through `slots.root` and `slotProps.root`. For example, `InputControl` lets you swap the outer container while keeping the input element intact:

```jsx
<InputControl
  slots={{ root: CustomContainer }}
  slotProps={{ root: { 'data-test': 'input-root' } }}
/>
```
