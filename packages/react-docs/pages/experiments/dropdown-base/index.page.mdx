# DropdownBase

A foundational dropdown component that renders a menu with selectable items, supporting dynamic content rendering and custom selection behavior.

## Examples

### Basic

* `defaultValue` / `value`: Use `defaultValue` for uncontrolled mode, or `value` paired with `onChange` for controlled mode.
* `items`: An array of selectable items. Each item typically includes a `label`, along with an optional identifier such as `value` of any type.
* `onChange`: Called with the selected item whenever the selection changes.
* `renderToggle`: Customizes the toggle UI.

```jsx
// Only the `label` property is required. Other fields are optional.
const items = [
  { id: 'all', label: 'All' },
  { id: 1, label: 'Item 1' },
  { id: 2, label: 'Item 2' },
];

const [value, setValue] = useState(items[0]);

<DropdownBase
  defaultValue={items[0]} // [optional] Initial value
  items={items}
  onChange={setValue}
  renderToggle={({ getToggleProps, value, renderItem }) => {
    const toggleProps = {
      disabled: false,
    };
    return (
      <MenuButtonToggle {...getToggleProps(toggleProps)}>
        {renderItem(value)}
      </MenuButtonToggle>
    );
  }}
/>
```

### Content sizing behavior

- `{toggleWidth}` is the toggle's `offsetWidth`
- `{contentWidth}` is `slotProps.content.width`

| Behavior                    | minWidth        | width                               | maxWidth | maxHeight | zIndex  |
| :-------------------------- | :-------------- | :---------------------------------- | :------- | :-------- | :------ |
| `matchWidth`                | —               | `{toggleWidth}` (inline via Popper) | 640px    | 40vh      | —       |
| `matchWidth` + `portalled`  | —               | `{toggleWidth}` (inline via Popper) | 640px    | 40vh      | popover |
| Default                     | `{toggleWidth}` | `{contentWidth} ?? 'max-content'`   | 640px    | 40vh      | —       |
| Default + `portalled`       | `{toggleWidth}` | `{contentWidth} ?? 'max-content'`   | 640px    | 40vh      | popover |

> Note: `maxHeight: '40vh'` follows MUI's approach. It scales with the viewport to avoid overflow on small screens while making better use of space on larger screens. A fixed height would either clip content or waste space.

## Props

The `DropdownBaseProps` interface defines the following properties for the `DropdownBase` component:

```ts
import { ReactNode, ReactElement, Ref } from 'react';

type DropdownItem = {
  type?: 'custom' | 'divider' | 'group' | 'submenu' | string;
  label?: ReactNode;
  children?: DropdownItem[];
  props?: Record<string, any>;
};

export interface DropdownBaseProps {
  /**
   * The content inside the dropdown toggle.
   * Can be a React node or a function that returns a React node.
   */
  children?: ((args: { getToggleProps: () => any }) => ReactNode) | ReactNode;

  /**
   * Callback triggered when an item is selected.
   * Receives the full `item` object, allowing access to custom fields or props.
   */
  onChange?: (item: DropdownItem) => void;

  /**
   * Array of items to be rendered in the dropdown menu.
   * Each item typically includes a `label`, and optionally a `type`, `props`, and `children`.
   */
  items?: DropdownItem[];

  /**
   * Optional custom render function for rendering individual items in the dropdown.
   * Defaults to rendering the `label` or `value` of the item.
   */
  renderItem?: (item: DropdownItem) => ReactNode;

  /**
   * Optional custom function to render the entire dropdown's content.
   * Provides access to `items`, `renderItem`, and `renderItems` for full customization.
   */
  renderContent?: (args: {
    items: DropdownItem[];
    renderItem: (item: DropdownItem) => ReactNode;
    renderItems: (items: DropdownItem[], prefix?: string) => ReactNode[];
  }) => ReactNode;

  /**
   * If `true`, sizes the dropdown list to match the toggle's `offsetWidth`.
   */
  matchWidth?: boolean;

  /**
   * Optional slots to override internal components like the toggle.
   * Example: `{ toggle: Toggle }`
   */
  slots?: {
    toggle?: React.ComponentType<any>;
  };

  /**
   * Optional slotProps to pass props to internal slot components.
   * Example: `{ toggle: { disabled: true } }`
   */
  slotProps?: {
    toggle?: Record<string, any>;
  };
}

/**
 * DropdownBase component.
 * A base component for rendering dropdown functionality with customizable options.
 */
declare const DropdownBase: React.ForwardRefExoticComponent<
  DropdownBaseProps & { ref?: Ref<any> }
>;

export default DropdownBase;
```

Each dropdown item is an object that may include the following fields:

* `label` (optional): The label to display for the item.
* `type` (optional): Specifies the type of item. It can be:
  * A regular menu item when the type is not provided.
    ```js
    {
      value: 'settings',
      label: 'Settings'
    }
    ```
  * `'custom'`: A custom element for a specific purpose.
    ```js
    {
      type: 'custom',
    }
    ```
  * `'divider'`: A divider that visually separates different groups of items.
    ```js
    {
      type: 'divider',
    }
    ```
  * `'group'`: A group of items, which may include nested items under `children`.
    ```js
    {
      type: 'group',
      label: 'Fruits',
      children: [
        { value: 'apple', label: 'Apple' },
        { value: 'banana', label: 'Banana' },
      ]
    }
    ```
  * `'submenu'`: Creates a nested menu that appears next to the item. You must provide a `children` array with more items. This is useful for grouping related items.
    ```js
    {
      type: 'submenu',
      label: (
        <Flex
          alignItems="center"
          columnGap="2x"
          justifyContent="space-between"
          width="100%"
        >
          File
          <AngleRightIcon />
        </Flex>
      ),
      children: [
        { value: 'new', label: 'New File' },
        { value: 'open', label: 'Open File' },
        { value: 'save', label: 'Save File' },
      ]
    }
    ```
* `props` (optional): Props passed directly to the underlying `MenuItem`, such as `disabled`, custom data attributes, or style props (`sx`).
* `[key: string]: any` (optional): Additional custom fields for your internal use, which are passed along in the `onChange` callback but are not used internally by the component.

### DropdownBase

| Name | Type | Default | Description |
| :--- | :--- | :------ | :---------- |
| onChange | `(item: DropdownItem) => void` | | Callback triggered when an item is selected. Receives the full `item` object, allowing you to access any custom fields or props defined for the item. |
| items | `Array<DropdownItem>` | [] | An array of items to be rendered in the dropdown menu. Each item should include a `value` (required), and optionally a `label`, `type`, `props`, and `children`. |
| matchWidth | boolean | `false` | If `true`, sizes the dropdown list to match the toggle's `offsetWidth`. See [Content sizing](#content-sizing-behavior) for the full matrix. |
| renderItem | `(item: DropdownItem) => ReactNode` | `defaultRenderItem` <sub>[1]</sub> | Optional custom render function for rendering dropdown items. |
| renderContent | `(args: { items: DropdownItem[], renderItem: (item: DropdownItem) => ReactNode, renderItems: (items: DropdownItem[], prefix?: string) => ReactNode[] }) => ReactNode` | | Optional custom function to render the entire dropdown's content. Provides access to `items`, `renderItem`, and `renderItems`. Use this to fully customize the dropdown content beyond the default rendering behavior. |
| slotProps	| `{ toggle?: Record<string, any>; content?: Record<string, any> }` | \{\} | The props used for each slot inside the `DropdownBase`. |

1. `defaultRenderItem` is the default render function for rendering dropdown items. It renders the item's `label` if the item is a plain object; otherwise, it renders the item directly.
    ```js
    const defaultRenderItem = (item) => isPlainObject(item) ? item.label : item;
    ```
