# DropdownBase

A foundational dropdown component that renders a menu with selectable options, supporting dynamic content rendering and custom selection behavior.

## Usage

Here's the simplest configuration:

```jsx
<DropdownBase
  onSelect={(option) => {
    setOption(option);
  }}
  options={[
    { value: 1, label: 'Option 1' },
    { value: 2, label: 'Option 2' },
  ]}
>
  {({ getToggleProps }) => {
    const { sx, ...restToggleProps } = getToggleProps();
    return (
      <Button
        {...restToggleProps}
        variant="secondary"
        sx={sx}
      >
        {option.label}
      </Button>
    );
  }}
</DropdownBase>
```

```js
toggleProps={{
  sx: {
    maxWidth: '100%',
    width: '100%',
    '> :first-of-type': {
      // Override the default `minWidth: auto` for flex items to enable text truncation
      minWidth: 0,
    },
  },
}}
```

## Props

The `DropdownBaseProps` interface defines the following properties for the `DropdownBase` component:

```ts
import { ReactNode, ReactElement, Ref } from 'react';

type Option = {
  value: any;
  type?: 'divider' | 'group' | 'submenu' | string;
  label?: ReactNode;
  children?: Option[];
  props?: Record<string, any>;
};

export interface DropdownBaseProps {
  children?: ((args: { getToggleProps: () => any }) => ReactNode) | ReactNode;
  onSelect?: (option: Option) => void;
  options?: Option[];
  renderOption?: (option: Option) => ReactNode;
  renderContent?: (args: {
    options: Option[];
    renderOption: (option: Option) => ReactNode;
    renderOptions: (options: Option[], prefix?: string) => ReactNode[];
  }) => ReactNode;
  toggleProps?: Record<string, any>;
}

declare const DropdownBase: React.ForwardRefExoticComponent<
  DropdownBaseProps & { ref?: Ref<any> }
>;

export default DropdownBase;
```

### `Option` object

Each option is an object that may include the following fields:

* `value` (required): A unique identifier or payload associated with the option.
* `label` (optional): The label to display for the option. If not provided, the `value` will be used.
* `type` (optional): Specifies the type of option. It can be:
  * (empty): A regular menu item.
    ```js
    {
      value: 'settings',
      label: 'Settings'
    }
    ```

  * `'divider'`: A divider that visually separates groups of items.
    ```js
    {
      type: 'divider',
    }
    ```
  * `'group'`: A group of options, which may contain nested options under `children`.
    ```js
    {
      type: 'group',
      label: 'Fruits',
      children: [
        { value: 'apple', label: 'Apple' },
        { value: 'banana', label: 'Banana' },
      ]
    }
    ```
  * `'submenu'`: Creates a nested menu that appears next to the option. You must provide a `children` array with more options. This is useful for grouping related items.
    ```js
    {
      type: 'submenu',
      label: (
        <Flex
          alignItems="center"
          columnGap="2x"
          justifyContent="space-between"
          width="100%"
        >
          File
          <AngleRightIcon />
        </Flex>
      ),
      children: [
        { value: 'new', label: 'New File' },
        { value: 'open', label: 'Open File' },
        { value: 'save', label: 'Save File' },
      ]
    }
    ```
* `props` (optional): Props passed directly to the underlying `MenuItem`, such as `disabled`, custom data attributes, or style props (`sx`).
* `[key: string]: any` (optional): Additional custom fields for your internal use, which are passed along in the `onSelect` callback but are not used internally by the component.

### Prop definitions

| Name | Type | Description |
| :--- | :--- | :---------- |
| children | ReactNode | Content inside the dropdown button. Accepts a React node or a render function. |
| onSelect | `(option: Option) => void` | Callback triggered when an option is selected. Receives the full `option` object, allowing you to access any custom fields or props defined for the option. |
| options | `Array<Option>` | An array of options to be rendered in the dropdown menu. Each option should include a `value` (required), and optionally a `label`, `type`, `props`, and `children`. |
| renderOption | `(option: Option) => ReactNode` | Optional custom render function for individual dropdown options. If not provided, the default function renders the `label` (if available) or `value` of the option. |
| renderContent | `(args: { options: Option[], renderOption: (option: Option) => ReactNode, renderOptions: (options: Option[], prefix?: string) => ReactNode[] }) => ReactNode` | Optional custom function to render the entire dropdown's content. Provides access to `options`, `renderOption`, and `renderOptions`. Use this to fully customize the dropdown content beyond the default rendering behavior. |
| toggleProps | `Record<string, any>` | Additional props to be passed to the toggle component. These props will be merged with the props returned by `getToggleProps()`. |
